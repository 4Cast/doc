Database

Wakanda server has a built-in database called WakandaDB that makes creating apps fun and easy.

In this part we will create a simple Todo dataclass and secure access to it by using the visual Model Editor.

Go ahead and create a Todo dataclass:

In terms of security we need the following :
  • Only a logged in user can Read/Create/Remove/Update Todos
  • A user can only create Todos for himself
  • A user can only Update/Remove/Read his own Todos

To assign permissions to the Todo dataclass, select the group todoUsers from the dropdown menu next to each action in the Properties tab of the Data Model Editor:

What we just did, will ensure that only logged in users who belong to the todoUsers group will have access to the dataclass Todo's data. But let's take this a step further, and restrict each user access to his own created Todos and stop a user from creating a Todo for another user !

To do so, we will make use off dataclass events and more precisely the save and restrict dataclass events.
