Web - Login Interface

In this section we will transform our empty loginComponent into a real world login interface !
  • The html and css of the whole web application will be based on the TodoMVC project.
  • For the sake of simplicity, add the TodoMVC css to the global css file ./src/style.css. This way the css will be shared by all the application components.

Our login component is composed of a template file (html), a class definition (ts), a styling file (css) and a unit test file (spec.ts).

Let's break down What we did here :

  • Import Wakanda service.
  • Add it to the login component providers list.
  • Assign it to a member of our component’s class.
  • Call the directory method login() to check if the user exists in the directory.
  • Call the method router navigate to redirect the user to the Todos view if the login was successful, otherwise we display an error message.
  • On the html side we call the login() method when the button is clicked and we bind the inputs to login and password.

Go to your browser and enter the username and password, the user will be redirected to TodoComponent when login succeeds and an error message appears when it fails.

That’s it … Congratulations your authentication system is alive !
