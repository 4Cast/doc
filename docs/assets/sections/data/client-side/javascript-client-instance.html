Getting the datastore object

This is not needed for authentication requests, since a WakandaClient instance is all you need for that use case.
Since the dataclasses list is generated dynamically during runtime, you need to call the getCatalog method with an optional list of dataclasses you want to use. If you don't specify any dataclasses you'll get all dataclasses the current user has access to.
In all following code examples, the ds variable refers to the initialized datastore object we've just retrieved and wakanda to an instance of WakandaClient.
In future versions, you'll have the choice to make the list of dataclasses static and avoid doing this initialization step.
