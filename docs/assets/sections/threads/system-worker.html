System Worker

Why

  • You need to run an external application or command line tool.
  • You need to retrieve the execution output of an external application.
  • You need to do it in a blocking or non blocking way.

What is it

  • A system worker runs external applications and gives you access to their output.
  • A system worker is run in its own thread.
  • You can get the output synchronously or asynchronously.
