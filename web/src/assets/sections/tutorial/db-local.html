Wakanda DB

Wakanda ships with an embedded database WakandaDB, the aim is allowing users to develop web applications as quickly as possible.

In this part we will create a simple Todo dataclass and implement access rights to the todoUsers group.

Creating a model is made easy with Wakanda since it provides a visual data editor.

Go ahead and create a Todo class:

Now let’s add permissions to our datastore : the group todoUsers will be granted the following privileges :

  • Read : Allows displaying the class entities.
  • Create : Add new Entity to our Todo class.
  • Update : Allows updating an entity.
  • Remove : Remove an entity from our Todo datastore.
  • Describe : accessing the class description.

To assign permissions to the Todo class, select the group todoUsers from the dropdown menu next to each action in the Properties tab of the Data Model Editor:

In the next section we’ll be enabling the CRUD functionalities. But before, we should mention datastore events that are called automatically by the server based on user actions.

In our case we will take advantage of events to enforce security and make sure only the allowed users are authorized to list and save Todos.

to check the complete list of events please refer to the Official Documentation

For now we are interested in save and restrict events

Thanks to this two events, we are reassured only the current user has the right to save and read from our datastore.
