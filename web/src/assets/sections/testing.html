Testing

From v2.1, Wakanda provides a way to write & run unit tests for your backend code:

  • It uses the standard Mocha/Chai modules.
  • It provides BDD test interface by default.
  • It runs all unit tests inside a NodeWorker.
Solutions prior to v2.1 need to follow the migration steps down there to enable the Test feature.

How to

  • Open the /tests/example.spec.js inside your backend project.
  • Write your unit tests in example.spec.js
  • You can create multiple JS files. There will be all runned as long as there are located in /tests/ folder.
  • Click Test button from the toolbar.
  • All test results are displayed in the Test console tab.
  • A report is written in tests/report/report.json.

CLI

Helpfull if you need to integrate your solution tests inside a continuous integration (CI) process.
  • Linux & Mac: $ wakanda-server path/to/solution.waSolution --test
  • Linux & Mac: $ wakanda-server path/to/solution.waSolution -t
  • Windows: > wakanda-server.exe path/to/solution.waSolution --test
  • Windows: > wakanda-server.exe path/to/solution.waSolution -t
It will:
  • Start Wakanda server
  • Load the given solution
  • Run all backend tests of the solution
  • Write a report in tests/report/report.json
  • Display a quick report in the stdout
