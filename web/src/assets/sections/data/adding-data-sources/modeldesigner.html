Use the Model Designer

Once you have filled the properties on the dialog, and clicked OK the model designer displays the remote schema with its tables, attributes, relations.
  • For remote relational databases, the relations displayed are based on Foreign key declarations.
  • The Primary key icon is dislayed next to the attribute.
  • The indexed attributes are in bold.
Note :When you are using an external database, you can still use the model editor to handle events and set CRUD permissions but you can't add/remove storage attributes to/from DataClasses.
You can however add virtual relation attributes, DataClass/Entity methods or calculated attributes via the Model API
